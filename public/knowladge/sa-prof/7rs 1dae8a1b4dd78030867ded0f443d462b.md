# 7rs

Okay, let's break down those seven Rs of cloud migration from the AWS blog. Here's a summary in markdown format to help you grasp the key concepts for the exam:

## **The Seven Rs of Cloud Migration**

These are the seven common strategies an organization can take when migrating applications and infrastructure to the cloud. Understanding these is crucial for the AWS Solution Architect Professional exam.

### **1. Retire**

- **What it means:** Turning off or decommissioning services and applications that are no longer needed.
- **Benefits:**
    - Reduces the attack surface.
    - Cost savings (potentially 10-20%).
    - Focuses resources on essential systems.

### **2. Retain**

- **What it means:** Deciding to keep certain resources on-premises and not migrate them to the cloud. This is a valid part of a cloud migration strategy.
- **Reasons:**
    - Security requirements.
    - Data compliance regulations.
    - Performance considerations.
    - Unresolved dependencies.
    - Lack of business value in migrating.
    - Migration complexity.

### **3. Relocate**

- **What it means:** Moving applications to their cloud-native version or moving EC2 instances between different VPCs, accounts, or AWS Regions without significant changes.
- **Example:** Migrating on-premises VMware workloads to VMware Cloud on AWS. The underlying architecture remains largely the same.

### **4. Rehost (Lift and Shift)**

- **What it means:** Migrating applications, databases, and data to AWS without making significant architectural changes. Essentially, you're moving your existing infrastructure to EC2 instances.
- **Characteristics:**
    - Simple and quick migration.
    - No cloud optimization is done initially.
    - Leverages cloud resources.
    - Potential cost savings (up to 30%) simply by using AWS infrastructure.
    - AWS Application Migration Service facilitates this.

### **5. Replatform (Lift and Reshape)**

- **What it means:** Making some cloud optimizations while migrating, without fundamentally changing the core application architecture.
- **Examples:**
    - Migrating a database to Amazon RDS.
    - Moving an application to AWS Elastic Beanstalk.
- **Benefits:**
    - Leverages managed services for benefits like backups, resiliency, and high availability.
    - Saves time and money through managed or serverless services.

### **6. Repurchase (Drop and Shop)**

- **What it means:** Moving to a completely different product, often a Software-as-a-Service (SaaS) solution, during the cloud migration.
- **Characteristics:**
    - Potentially expensive in the short term.
    - Very quick to deploy.
- **Examples:**
    - Moving CRM to Salesforce.com.
    - Migrating HR systems to Workday.
    - Switching CMS to Drupal Cloud.

### **7. Refactor / Re-architect**

- **What it means:** Re-imagining and re-architecting the application using cloud-native features to improve scalability, performance, security, and agility.
- **Characteristics:**
    - Most effort involved.
    - Highest potential payoff in leveraging cloud capabilities.
    - Often involves breaking down monolithic applications into microservices.
- **Examples:**
    - Migrating an application to a serverless architecture.
    - Using Amazon S3 for data storage.

Understanding these seven strategies and when to apply them is key for the AWS Solution Architect Professional exam. Make sure to consider the trade-offs and benefits of each approach in different scenarios.
